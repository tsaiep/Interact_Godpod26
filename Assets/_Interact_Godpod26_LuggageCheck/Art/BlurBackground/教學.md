# URP Fullscreen Shader 串接 Volume 教學

這份文件記錄如何把 Unity URP 官方 fullscreen Renderer Feature 範例，改成可以被 Volume 控制的 fullscreen post-processing effect。

範例對應檔案：

- `BlurWeightControlRendererFeature.cs`
- `BlurWeightControlVolumeComponent.cs`
- `SH_BlurBackground.shadergraph`
- `M_BlurBackground.mat`

## 目標

要達成的行為：

1. Renderer Feature 只在 Volume 啟用時執行。
2. Volume Profile 裡的 `Intensity` 和 Volume 本身的 `Weight` 都要有效。
3. Fullscreen Shader Graph 可以接收 camera color 作為來源圖。
4. 如果 fullscreen shader 使用 stencil，只讓通過 stencil 的像素套效果。
5. 沒有通過 stencil 的像素要保留原本畫面，不能變黑。

## 官方範例需要改的地方

官方 fullscreen Renderer Feature 範例通常只示範：

- 建立 `ScriptableRendererFeature`
- 建立 `ScriptableRenderPass`
- 建立 temporary color texture
- 用 material 畫 fullscreen triangle

但要串成真正可用的 Volume effect，還需要補以下幾件事。

## 1. 建立 Volume Component

Volume Component 要繼承 `VolumeComponent`，並實作 `IPostProcessComponent`。

關鍵是 Volume Parameter 的預設值要是「未套效果」的狀態，否則 Volume weight 會看起來沒有作用。

```csharp
public sealed class BlurWeightControlVolumeComponent : VolumeComponent, IPostProcessComponent
{
    public ClampedFloatParameter intensity = new ClampedFloatParameter(0f, 0f, 1f);

    public bool IsActive()
    {
        return intensity.GetValue<float>() > 0.0f;
    }
}
```

不要把預設值設成 `1f`：

```csharp
public ClampedFloatParameter intensity = new ClampedFloatParameter(1f, 0f, 1f);
```

原因是 Volume 系統會用 Volume weight 在預設值和 override 值之間混合。  
如果預設值是 `1`，override 也是 `1`，那 weight 從 0 到 1 都會得到 `1`，看起來就像 Volume weight 完全沒作用。

正確概念：

- default value = 效果關閉狀態，例如 `0`
- override value = Volume 裡想套用的效果強度，例如 `1`
- Volume weight = 從 default value 混到 override value 的權重

## 2. Renderer Feature 裡讀取 Volume

在 `AddRenderPasses` 裡讀取目前 camera 的 Volume stack。

```csharp
BlurWeightControlVolumeComponent myVolume =
    VolumeManager.instance.stack?.GetComponent<BlurWeightControlVolumeComponent>();

if (myVolume == null || !myVolume.IsActive())
    return;
```

這段決定 pass 是否要 enqueue。

```csharp
renderer.EnqueuePass(m_FullScreenPass);
```

如果沒有這段檢查，Renderer Feature 會一直執行，不會像一般 post-processing 一樣受 Volume 控制。

## 3. 把 Volume 值傳給 Shader

官方範例通常只示範傳 `_BlitTexture` 和 `_BlitScaleBias`。  
要讓 Volume 控制 shader，必須額外把 Volume 參數送進 material。

建議先快取 shader property id：

```csharp
private static readonly int kBlitTexturePropertyId = Shader.PropertyToID("_BlitTexture");
private static readonly int kBlitScaleBiasPropertyId = Shader.PropertyToID("_BlitScaleBias");
private static readonly int kIntensityPropertyId = Shader.PropertyToID("_Intensity");
private static readonly int kAlphaPropertyId = Shader.PropertyToID("_Alpha");
```

在 draw 前設定：

```csharp
BlurWeightControlVolumeComponent myVolume =
    VolumeManager.instance.stack?.GetComponent<BlurWeightControlVolumeComponent>();

if (myVolume != null)
{
    float intensity = myVolume.intensity.value;
    s_SharedPropertyBlock.SetFloat(kIntensityPropertyId, intensity);
    s_SharedPropertyBlock.SetFloat(kAlphaPropertyId, intensity);
}
```

這裡同時設定 `_Intensity` 和 `_Alpha` 是為了兩種 shader 寫法：

- 如果 shader 有 `_Intensity`，可以直接拿它做 lerp。
- 如果 shader graph 用 fullscreen alpha blend，可以把 Volume intensity 接到 `_Alpha`。

重點：C# 設的 property name 必須跟 Shader Graph exposed reference name 一致。  
如果 Shader Graph 裡沒有 `_Intensity`，C# 設 `_Intensity` 不會產生任何效果。

## 4. Shader Graph 要真的使用 Volume 值

只在 C# 裡設定 `_Alpha` 或 `_Intensity` 還不夠，Shader Graph 裡必須真的接線使用。

目前這個範例的做法：

1. Shader Graph 有一個 exposed float property：`_Alpha`
2. `_Alpha` 接到 Fragment 的 `Alpha`
3. Fullscreen target 的 Blend Mode 設成 `Alpha`

這樣 Volume intensity 才能用 alpha blending 淡入淡出 fullscreen effect。

如果沒有把 `_Alpha` 接到 Fragment Alpha，Volume weight 不會影響畫面。  
如果 Blend Mode 是 `Disabled`，即使 Alpha block 有值，也不會拿來混合 color。

Shader Graph 設定重點：

- Graph Target：Fullscreen
- Blend Mode：Alpha
- `_Alpha` property：Exposed
- `_Alpha` 接到 Fragment Alpha
- color output 是你的效果結果

另一種寫法是在 Shader Graph 內做 lerp：

```text
FinalColor = lerp(OriginalColor, EffectColor, _Intensity)
```

這種方式可以不用依賴 alpha blend，但 shader 必須能同時取到原圖和效果圖。

## 5. Fullscreen Shader 要讀 camera color

Renderer Feature 會把當前 camera color 傳給 shader：

```csharp
if (sourceTexture != null)
    s_SharedPropertyBlock.SetTexture(kBlitTexturePropertyId, sourceTexture);

s_SharedPropertyBlock.SetVector(kBlitScaleBiasPropertyId, new Vector4(1, 1, 0, 0));
```

Shader Graph 或 HLSL 需要使用 `_BlitTexture` 作為原畫面來源。

如果使用 Unity fullscreen/blit 相關節點或 `Blit.hlsl`，通常也需要 `_BlitScaleBias`，避免 UV 翻轉或縮放錯誤。

## 6. 開啟 stencil 測試

如果 Shader Graph 裡有 stencil 設定，Renderer Feature 還必須綁定 depth-stencil attachment。

```csharp
private static readonly bool kBindDepthStencilAttachment = true;
```

RenderGraph pass 裡：

```csharp
builder.SetRenderAttachmentDepth(resourcesData.activeDepthTexture, AccessFlags.Read);
```

這樣 shader pass 裡的 stencil test 才會有 stencil buffer 可以讀。

這裡用 `AccessFlags.Read`，因為這個 pass 只需要讀 stencil 來測試，不需要改寫 depth 或 stencil。

## 7. Stencil 沒通過的像素不能變黑

這是官方範例最容易踩到的坑。

如果流程是：

1. 建立一張新的 temporary color texture
2. 直接用 fullscreen material 畫進去
3. shader 有 stencil test

那 stencil 沒通過的像素就不會被 shader 寫入。  
temporary texture 原本沒有有效顏色，所以那些像素可能是黑色或未定義資料。

解法是先把原本畫面 copy 到 temporary texture，再用 stencil pass 覆蓋通過區域。

流程：

```text
cameraColor
  -> copy 到 destination
  -> 用 fullscreen material 畫 destination
  -> stencil 通過的像素被效果覆蓋
  -> stencil 沒通過的像素保留 copy 過來的原畫面
  -> resourcesData.cameraColor = destination
```

RenderGraph copy pass：

```csharp
using (var builder = renderGraph.AddRasterRenderPass<CopyPassData>(
    $"{passName} Copy Color", out var passData, profilingSampler))
{
    passData.inputTexture = source;

    builder.UseTexture(passData.inputTexture, AccessFlags.Read);
    builder.SetRenderAttachment(destination, 0, AccessFlags.WriteAll);
    builder.SetRenderFunc((CopyPassData data, RasterGraphContext context) =>
        ExecuteCopyColorPass(data, context));
}
```

Main pass：

```csharp
builder.UseTexture(passData.inputTexture, AccessFlags.Read);
builder.SetRenderAttachment(destination, 0, AccessFlags.ReadWrite);
builder.SetRenderAttachmentDepth(resourcesData.activeDepthTexture, AccessFlags.Read);
```

`AccessFlags.ReadWrite` 很重要。  
因為 stencil test 會讓部分像素不被寫入，這時 render target 必須保留 copy pass 已經寫好的顏色。

不要用：

```csharp
builder.SetRenderAttachment(destination, 0, AccessFlags.Write);
```

`Write` 只表示這個 pass 會寫入部分資料，不保證保留原本 destination 的內容。

## 8. 設定 cameraColor 給後續 pass 使用

套完效果後要更新 URP 的 camera color resource：

```csharp
resourcesData.cameraColor = destination;
```

否則後面的 render pass 可能還是讀舊的 camera color。

## 9. Material 要換成自己的 fullscreen shader

官方範例常用這個預設材質：

```csharp
m_Material = UnityEditor.AssetDatabase.LoadAssetAtPath<Material>(
    "Packages/com.unity.render-pipelines.universal/Runtime/Materials/FullscreenInvertColors.mat");
```

這只是範例用的 invert color material。  
正式效果要換成自己的 material，例如 `M_BlurBackground.mat`。

可以有兩種做法：

1. 在 Renderer Feature inspector 裡序列化指定 material。
2. 在 `Create()` 裡用 `AssetDatabase.LoadAssetAtPath` 指到自己的 material。

例如：

```csharp
m_Material = UnityEditor.AssetDatabase.LoadAssetAtPath<Material>(
    "Assets/_Interact_Godpod26_LuggageCheck/Art/BlurBackground/M_BlurBackground.mat");
```

如果欄位有 `[HideInInspector]`，Inspector 不會顯示 material 欄位，比較不方便調整。  
正式工具可以考慮移除 `[HideInInspector]`。

## 10. Render Pass 時機

目前範例使用：

```csharp
m_FullScreenPass.renderPassEvent = RenderPassEvent.AfterRenderingPostProcessing;
```

這代表效果會在 URP 內建 post-processing 之後執行。

常見選擇：

- `BeforeRenderingTransparents`：影響透明物件前的畫面，不適合所有 fullscreen post effect。
- `BeforeRenderingPostProcessing`：在 URP post-processing 前套效果。
- `AfterRenderingPostProcessing`：在 URP post-processing 後套效果。

如果你的 shader 要讀最終畫面，通常用 `AfterRenderingPostProcessing`。

## 11. 必要檢查清單

串接失敗時照這份檢查：

- Renderer Data 有加上 `BlurWeightControlRendererFeature`。
- Camera 使用的 URP Renderer Data 是同一份。
- Scene 裡有 Global Volume 或 Local Volume。
- Volume Profile 有加 `BlurWeightControl` override。
- `Intensity` 有勾 override checkbox。
- `Intensity` override 值大於 `0`。
- Volume 本身的 `Weight` 大於 `0`。
- Volume layer mask 和 Camera volume mask 對得到。
- Renderer Feature 的 material 是自己的 fullscreen shader material，不是官方 invert color material。
- Shader Graph exposed reference name 和 C# property name 一致。
- Shader Graph 的 `_Alpha` 有接到 Fragment Alpha。
- Shader Graph 的 Fullscreen Blend Mode 是 `Alpha`。
- 如果用 stencil，Shader Graph stencil 設定已開。
- 如果用 stencil，Renderer Feature 有 `SetRenderAttachmentDepth(resourcesData.activeDepthTexture, AccessFlags.Read)`。
- 如果 stencil failed pixels 變黑，確認有先 copy camera color，main pass color attachment 是 `ReadWrite`。

## 最小成功版本架構

成功串接 Volume fullscreen shader 的最小結構如下：

```text
VolumeComponent
  - ClampedFloatParameter intensity default = 0
  - IsActive() 回傳 intensity > 0

RendererFeature.AddRenderPasses
  - 從 VolumeManager 讀 component
  - component inactive 就 return
  - enqueue fullscreen pass

RenderGraph
  - source = resourcesData.cameraColor
  - destination = temp texture
  - copy source -> destination
  - main pass 讀 source，寫 destination
  - main pass 綁 activeDepthTexture 給 stencil
  - main pass color attachment 用 ReadWrite
  - resourcesData.cameraColor = destination

Shader Graph
  - Fullscreen target
  - 讀 _BlitTexture
  - 用 _Alpha 或 _Intensity 控制效果強度
  - Blend Mode = Alpha，或 shader 內自己 lerp
  - stencil 需要時在 graph target 裡開啟
```

## 這次實作改掉的重點

`BlurWeightControlVolumeComponent.cs`

- `intensity` default 從 `1f` 改成 `0f`
- 讓 Volume weight 可以正確從無效果混到有效果

`BlurWeightControlRendererFeature.cs`

- 開啟 depth-stencil attachment
- 把 Volume intensity 傳到 `_Intensity` 和 `_Alpha`
- 加入 copy pass，先保存原始 camera color
- main pass 改用 `AccessFlags.ReadWrite`
- stencil 沒通過的像素保留原畫面

`SH_BlurBackground.shadergraph`

- Fullscreen Blend Mode 改成 `Alpha`
- `_Alpha` 接到 Fragment Alpha
- stencil 保持開啟
