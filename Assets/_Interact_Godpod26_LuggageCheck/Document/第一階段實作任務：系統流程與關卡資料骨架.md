# 第一階段實作任務：系統流程與關卡資料骨架

請先閱讀完整的《Unity RFID 行李安檢互動系統規格書》，理解最終系統架構與互動流程。

本階段不要實作完整功能，只建立後續功能需要依賴的系統流程、狀態機及關卡資料架構。

---

## 1. 本階段目標

建立一套可擴充的系統骨架，使 Unity 專案可以：

1. 進入待機狀態。
2. 透過測試按鍵選擇六個關卡之一。
3. 載入對應的關卡設定資料。
4. 按照指定流程切換系統狀態。
5. 模擬成功或失敗結果。
6. 完成重置並返回待機狀態。
7. 在 Console 清楚顯示目前狀態與關卡資訊。

本階段完成後，系統即使尚未接上真正影片、RFID、UI 與互動物件，也必須能完整走完一次模擬流程。

---

## 2. 本階段暫不實作

本階段不要實作以下內容：

* 真實 RFID 硬體通訊
* StreamingAssets 影片播放
* VideoPlayer 預載
* RenderTexture 切換
* 靜態背景圖片載入
* 行李物件移動
* 不規則物件方向選擇
* 違禁品正確或錯誤判斷
* 倒數計時 UI
* 違禁品提示 UI
* 粒子特效
* 正式音效
* 最終美術資源

上述功能只需保留可供後續接入的接口，不要在本階段製作假完整實作。

---

## 3. 專案架構原則

請遵守以下架構原則：

* 使用單一主場景。
* `GameFlowManager` 是唯一可以決定主要系統狀態的物件。
* 其他模組不得自行修改系統狀態。
* 關卡差異必須透過資料設定，而不是複製六套流程程式。
* 不可將所有功能集中在同一支腳本。
* 不可使用場景物件名稱作為主要邏輯判斷。
* 不可在每幀使用 `FindObjectOfType`、`GameObject.Find` 或類似搜尋。
* 不可自動修改規格範圍外的既有腳本。
* 不可刪除既有場景物件。
* 新增的公開 Inspector 欄位需有清楚的名稱或 Tooltip。
* 所有事件訂閱必須正確解除，避免重複觸發。

## 5. 建立系統狀態

建立 `GameState.cs`。

狀態至少包含：

```csharp
public enum GameState
{
    SystemInitializing,
    IdlePreparing,
    Idle,
    LevelInitializing,
    IntroPreparing,
    IntroPlaying,
    GamePreparing,
    Gameplay,
    SuccessPreparing,
    SuccessPlaying,
    FailurePreparing,
    FailurePlaying,
    Resetting,
    ErrorRecovery
}
```

本階段可使用所有正式狀態，但只以模擬方式完成流程。

---

## 6. 建立 LevelConfig

建立 `LevelConfig.cs`，型別為 `ScriptableObject`。

使用：

```csharp
[CreateAssetMenu(
    fileName = "LevelConfig",
    menuName = "RFID Baggage/Level Config"
)]
```

本階段至少包含以下欄位：

```csharp
string levelId;
string displayName;
string rfidId;

string introVideoRelativePath;
string successVideoRelativePath;
string failureVideoRelativePath;
string finalFrameImageRelativePath;

float gameplayDuration;
float inputCooldown;
bool wrongSelectionDeductsTime;
float wrongSelectionTimePenalty;
```

需求：

* 所有欄位使用 private serialized field。
* 提供唯讀 public property。
* `gameplayDuration` 預設為 15 秒。
* 不在 ScriptableObject 中保存執行期間狀態。
* 影片與圖片欄位保存 StreamingAssets 相對路徑，不保存絕對路徑。
* 加入基本資料驗證，避免關卡 ID、RFID ID 或時間設定無效。

本階段不需要實際讀取影片或圖片。

---

## 7. 建立 LevelDatabase

建立 `LevelDatabase.cs`，型別可使用 `ScriptableObject`。

功能：

* 保存六個 `LevelConfig`。
* 依 `levelId` 查找關卡。
* 依 `rfidId` 查找關卡。
* 檢查重複的 `levelId`。
* 檢查重複的 `rfidId`。
* 忽略空白與無效資料。
* 找不到關卡時回傳 false，不直接拋出未處理例外。

公開介面建議：

```csharp
public IReadOnlyList<LevelConfig> Levels { get; }

public bool TryGetByLevelId(
    string levelId,
    out LevelConfig level
);

public bool TryGetByRfidId(
    string rfidId,
    out LevelConfig level
);
```

RFID 查詢前需要移除前後空白及換行符號。

---

## 8. 建立 GameFlowManager

建立 `GameFlowManager.cs`。

此腳本負責：

* 保存目前系統狀態。
* 保存目前關卡。
* 驗證狀態切換是否合法。
* 接收開始關卡要求。
* 接收各階段完成通知。
* 接收成功與失敗通知。
* 執行重置並返回 Idle。
* 防止重複觸發。

公開唯讀屬性：

```csharp
public GameState CurrentState { get; }
public LevelConfig CurrentLevel { get; }
```

公開事件：

```csharp
public event Action<GameState, GameState> StateChanged;
public event Action<LevelConfig> LevelStarted;
public event Action<LevelConfig> GameplayStarted;
public event Action<LevelConfig, bool> LevelFinished;
public event Action LevelReset;
```

公開介面至少包含：

```csharp
public bool RequestStartLevelByRfid(string rfidId);
public bool RequestStartLevel(LevelConfig level);

public void NotifyLevelInitialized();
public void NotifyIntroPrepared();
public void NotifyIntroCompleted();
public void NotifyGamePrepared();

public void NotifyGameSuccess();
public void NotifyGameFailure();

public void NotifyResultPrepared();
public void NotifyResultCompleted();

public void ReportRecoverableError(string message);
public void ReturnToIdle();
```

方法名稱可依專案命名風格調整，但功能與責任需保持清楚。

---

## 9. 狀態切換規則

本階段需限制合法狀態切換。

主要流程如下：

```text
SystemInitializing
→ IdlePreparing
→ Idle

Idle
→ LevelInitializing
→ IntroPreparing
→ IntroPlaying
→ GamePreparing
→ Gameplay
```

成功流程：

```text
Gameplay
→ SuccessPreparing
→ SuccessPlaying
→ Resetting
→ IdlePreparing
→ Idle
```

失敗流程：

```text
Gameplay
→ FailurePreparing
→ FailurePlaying
→ Resetting
→ IdlePreparing
→ Idle
```

錯誤恢復流程：

```text
任一需要恢復的狀態
→ ErrorRecovery
→ Resetting
→ IdlePreparing
→ Idle
```

要求：

* `Idle` 以外的狀態不能開始新關卡。
* `Gameplay` 以外的狀態不能回報成功或失敗。
* 成功與失敗只能觸發其中一個。
* 同一通知重複呼叫時不得重複切換。
* 不合法的狀態切換需在 Console 顯示 Warning。
* 不合法操作不得讓系統進入無法恢復的狀態。
* 主要狀態只能由 `GameFlowManager` 寫入。

## 10. 建立 DebugLevelInput

建立 `DebugLevelInput.cs`。

用途是暫時模擬 RFID 與後續系統通知。

測試控制：

| 按鍵      | 功能                     |
| ------- | ---------------------- |
| 數字鍵 1～6 | 在 Idle 狀態下選擇對應關卡       |
| I       | 模擬關卡初始化完成              |
| P       | 模擬進場影片 Prepare 完成      |
| O       | 模擬進場影片播放完成             |
| G       | 模擬遊戲畫面準備完成並進入 Gameplay |
| S       | 在 Gameplay 狀態下模擬成功     |
| F       | 在 Gameplay 狀態下模擬失敗     |
| R       | 模擬結果影片準備完成             |
| E       | 模擬結果影片播放完成             |
| Escape  | 強制進入安全重置流程             |

數字鍵 1～6 不要直接呼叫固定流程，應透過 `LevelDatabase` 取得關卡，再向 `GameFlowManager` 提出開始要求。

請避免測試腳本繞過正式公開接口，直接修改 Manager 私有欄位。

如果專案使用 New Input System，請配合現有輸入架構；不要為了本測試自行更換專案輸入系統。

## 13. Console 紀錄

每次狀態切換需輸出：

```text
[GameFlow] Idle -> LevelInitializing
```

開始關卡時輸出：

```text
[GameFlow] Starting Level_01, RFID: RFID_01
```

成功或失敗時輸出：

```text
[GameFlow] Level_01 Result: Success
```

或：

```text
[GameFlow] Level_01 Result: Failure
```

重置完成時輸出：

```text
[GameFlow] Reset complete. Returned to Idle.
```

不合法操作需使用 Warning，不要直接造成 NullReferenceException。

## 14. 本階段驗收流程

### 測試一：成功流程

1. 啟動 Play Mode。
2. 系統進入 Idle。
3. 按數字鍵 1。
4. 系統載入 Level_01。
5. 依序按 I（模擬關卡初始化完成）、P（模擬進場影片準備完成）、O（模擬進場影片播放完成）、G（模擬遊戲畫面準備完成並進入 Gameplay）。
6. 系統進入 Gameplay。
7. 按 S。
8. 系統進入 SuccessPreparing。
9. 按 R。
10. 系統進入 SuccessPlaying。
11. 按 E。
12. 系統重置並返回 Idle。

### 測試二：失敗流程

1. 在 Idle 按數字鍵 2。
2. 依序按 I（模擬關卡初始化完成）、P（模擬進場影片準備完成）、O（模擬進場影片播放完成）、G（模擬遊戲畫面準備完成並進入 Gameplay）。
3. 在 Gameplay 按 F。
4. 按 R、E。
5. 系統返回 Idle。

### 測試三：非法輸入

需確認：

* 在 Idle 按 S 不會進入成功狀態。
* 在 IntroPlaying 按 F 不會進入失敗狀態。
* 關卡進行中按數字鍵 1～6 不會切換關卡。
* 連續按兩次 S 不會觸發兩次成功。
* 連續按兩次 E 不會重複重置。
* 無效 RFID 不會開始關卡。

### 測試四：重複遊玩

依序完成：

```text
Level_01 成功
Level_01 失敗
Level_03 成功
Level_06 失敗
```

每次結束後均需正確返回 Idle，且上一關卡資料不得殘留。
