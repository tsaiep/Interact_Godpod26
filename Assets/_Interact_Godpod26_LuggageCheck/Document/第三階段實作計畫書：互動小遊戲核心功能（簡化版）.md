# 2. 本階段目標

本階段實作互動小遊戲的核心功能。

系統進入 `Gameplay` 狀態後，需要完成：

1. 啟用目前關卡的行李物件。
2. 收集目前關卡中所有可選取物件。
3. 自動選取預設物件。
4. 使用方向鍵在不規則排列的物件間移動選取。
5. 使用空白鍵確認目前選取物件。
6. 判斷目前物件是否為違禁品。
7. 選擇正確時移除違禁品並更新下方提示 UI。
8. 選擇錯誤時播放錯誤回饋，但不移除物件。
9. 顯示倒數計時數字與進度。
10. 在時間內找完全部違禁品時判定成功。
11. 時間歸零且尚未完成時判定失敗。
12. 將成功或失敗回報給既有 `GameFlowManager`。
13. 關卡結束後重置物件、倒數、UI 與選取狀態。
14. 使用 UnityEvent 配置各物件與遊戲階段的視覺、音效及動畫回饋。

本階段完成後，完整流程應能實際操作：

```text
Idle
→ 選擇關卡
→ Intro
→ Gameplay
→ 方向鍵選取
→ 空白鍵確認
→ 正確或錯誤回饋
→ Success 或 Failure
→ Reset
→ Idle
```

# 3. 本階段不實作內容

本階段不要實作：

* 真實 RFID Reader
* Serial Port 或 HID 通訊
* 影片播放順暢度優化
* 多 VideoPlayer Ping-Pong 架構
* 行李內容物件與 Intro 影片的位置同步
* 正式美術特效
* 正式音效資產
* 排行榜
* 分數系統
* 難度系統
* 儲存進度
* 多人輸入
* 複雜關卡編輯工具
* 額外建立大量資料型 ScriptableObject

可以使用測試用圖片、Animator、ParticleSystem、AudioSource 或簡單 GameObject 來驗證流程。

沿用既有設計，負責：

* 管理主要系統狀態
* 保存目前關卡
* 驗證狀態切換是否合法
* 接收遊戲成功
* 接收遊戲失敗
* 進入 Success、Failure 與 Reset 流程

不得新增以下責任：

* 不直接讀取方向鍵

* 不直接管理單一行李物件
* 不直接判斷違禁品
* 不直接播放正確或錯誤特效

---

## 5.2 GameplayController

負責整個小遊戲規則。

包含：

* 監聽 `GameFlowManager` 狀態
* 進入 Gameplay 時初始化目前關卡
* 收集目前關卡物件
* 初始化違禁品 UI
* 啟動及更新倒數
* 接收玩家確認的物件
* 判斷正確或錯誤
* 更新違禁品完成狀態
* 更新倒數 UI
* 判斷成功或失敗
* 回報 `GameFlowManager`
* 在 Resetting 時重置全部 Gameplay 內容
* 觸發遊戲層級 UnityEvent

不需要再拆分：

* GameplayCoordinator
* ContrabandGameController
* CountdownController
* CountdownUIController
* ContrabandUIController
* GameplayFeedbackController
* LevelRuntimeController

## 5.3 ItemSelectionController

只負責選取操作。

包含：

* 接收方向鍵

* 將確認結果通知 `GameplayController`
* 方向輸入冷卻
* 確認輸入冷卻
* 啟用與停用輸入

不負責：

* 不判斷是否為違禁品

* 不直接呼叫 `GameFlowManager`

並提供 UnityEvent：

* 被選取

不負責：

* 不計算違禁品總數

# 8. 建立 SelectableItem

建立 `SelectableItem.cs`。

## 8.1 Inspector 欄位

至少包含：

```csharp
[SerializeField]
private string itemId;

[SerializeField]
private bool isContraband;

[SerializeField]
private int contrabandSlotIndex = -1;

[SerializeField]
private Transform selectionPoint;

[SerializeField]
private GameObject visualRoot;
```

說明：

* `itemId`：物件識別名稱。
* `isContraband`：是否為違禁品。
* `contrabandSlotIndex`：對應下方提示 UI 的索引。
* 非違禁品可設為 `-1`。
* `selectionPoint`：方向選取時計算位置。
* 若未設定，可使用自身 Transform。
* `visualRoot`：需要隱藏或恢復的物件顯示 Root。
* 若未設定，可使用目前 GameObject。

本專案不需要額外建立 `ItemConfig` ScriptableObject。

---

## 8.2 執行期屬性

至少提供：

```csharp
public string ItemId { get; }
public bool IsContraband { get; }
public int ContrabandSlotIndex { get; }
public Transform SelectionPoint { get; }
public bool IsSelectable { get; }
public bool IsResolved { get; }
public bool IsSelected { get; }
```

---

## 8.3 公開方法

建議：

```csharp
public void SetSelected(bool selected);
public void SetSelectable(bool selectable);
public void MarkCorrect();
public void PlayWrongFeedback();
public void ResetItem();
```

行為：

### SetSelected

* 更新目前選取狀態。
* `true` 時觸發 `onSelected`。
* `false` 時觸發 `onDeselected`。
* 相同狀態不得重複觸發事件。

### MarkCorrect

* 將 `IsResolved` 設為 true。
* 將 `IsSelectable` 設為 false。
* 取消選取。
* 觸發 `onCorrect`。
* 可由 UnityEvent 隱藏 `visualRoot` 或播放移除動畫。

### PlayWrongFeedback

* 不更改完成狀態。
* 不移除物件。
* 觸發 `onWrong`。

### ResetItem

* 恢復初始可選取狀態。
* 清除完成狀態。
* 清除選取狀態。
* 恢復 `visualRoot`。
* 觸發 `onReset`。

---

## 8.4 UnityEvent

加入：

```csharp

[SerializeField]

[SerializeField]
private UnityEvent onCorrect;

[SerializeField]
private UnityEvent onWrong;

[SerializeField]

```

建議用途：

### onSelected

* 開啟 Outline
* 啟用發光物件
* 顯示選取框
* 播放輕微縮放動畫

### onDeselected

* 關閉 Outline
* 關閉發光
* 隱藏選取框

### onCorrect

* 播放正確粒子
* 播放正確音效
* 播放縮小或淡出動畫
* 隱藏物件

### onWrong

* 播放錯誤粒子
* 播放錯誤音效
* 播放震動動畫
* 顯示錯誤符號

### onReset

* 重置 Animator
* 停止粒子
* 停止音效
* 恢復顯示狀態

UnityEvent 不可綁定：

* `NotifyGameSuccess`
* `NotifyGameFailure`
* 修改 GameState
* 增加找到數量
* 修改倒數核心時間

## 10.1 方向鍵規則

| 按鍵 | 方向        |
| -- | --------- |
| 上  | `(0, 1)`  |
| 下  | `(0, -1)` |
| 左  | `(-1, 0)` |

# 11. 不規則物件方向選取

## 11.1 使用螢幕座標

因為行李物件可能是 3D GameObject，但畫面以固定 2D 視角操作，因此使用：

```csharp
selectionCamera.WorldToScreenPoint(
    item.SelectionPoint.position
);
```

不要單純使用世界座標 X、Y、Z 判斷上下左右。

---

## 11.2 候選篩選

移動選取時：

1. 排除目前物件。
2. 排除 `IsSelectable == false`。
3. 排除 `IsResolved == true`。
4. 排除停用中的物件。
5. 排除位於攝影機後方的物件。
6. 計算候選物件相對於目前物件的螢幕方向。
7. 只保留位於輸入方向的候選。
8. 計算候選分數。
9. 選擇最低分物件。

方向判斷：

```csharp
float dot = Vector2.Dot(
    offset.normalized,
    inputDirection.normalized
);
```

只有：

```csharp
dot >= minimumDirectionDot
```

的候選物件才有效。

---

## 11.3 評分方式

建議簡化為：

```text
方向偏差越小越優先
距離越近越優先
```

可使用：

```text
Score =
(1 - dot) × angleWeight
+
normalizedDistance × distanceWeight
```

不需要在第一版加入：

* 遮擋判定
* NavMesh
* 自訂鄰接圖
* 手動上下左右連線
* 複雜路徑搜尋

如果某些特殊排列無法靠演算法得到理想結果，後續再考慮對單一物件加入手動鄰接覆寫，不要本階段先實作。

---

## 11.4 沒有候選物件

如果指定方向沒有有效物件：

* 保持目前選取
* 不清除選取
* 不播放錯誤音效
* 不觸發重複選取 UnityEvent

---

## 11.5 預設選取物件

Gameplay 開始時：

1. 優先使用關卡指定的預設物件。
2. 若沒有指定，選擇第一個有效物件。
3. 若沒有任何有效物件，回報初始化錯誤。

預設物件可使用：

* `LevelConfig` 直接引用
* 或 `GameplayController` 中的關卡資料設定

若跨 Scene 或 ScriptableObject 不適合直接引用場景物件，也可在 `SelectableItem` 增加：

```csharp
[SerializeField]
private bool defaultSelected;
```

每個關卡只能有一個預設選取物件。

# 16. BeginGameplay 流程

進入 Gameplay 後：

1. 確認 `CurrentLevel` 有效。
2. 找到目前關卡 Root。
3. 停用其他關卡 Root。
4. 啟用目前關卡 Root。
5. 收集目前 Root 下的 `SelectableItem`。
6. 清除無效或重複項目。
7. 呼叫每個物件的 `ResetItem()`。
8. 計算違禁品總數。
9. 驗證違禁品 UI Slot 數量。
10. 重置全部 Slot。
11. 將 `foundContrabandCount` 設為 0。
12. 將 `remainingTime` 設為關卡設定時間。
13. 顯示 Gameplay UI。
14. 更新倒數數字與進度。
15. 初始化 `ItemSelectionController`。
16. 選取預設物件。
17. 啟用輸入。
18. 將 `gameplayRunning` 設為 true。
19. 觸發 `onGameplayStarted`。

若初始化失敗：

* 不啟動倒數
* 不啟用輸入
* 輸出明確錯誤
* 呼叫既有錯誤恢復接口

---

# 17. 倒數計時

倒數直接由 `GameplayController.Update()` 管理。

只在：

```csharp
gameplayRunning == true
```

時更新。

計算：

```csharp
remainingTime -= Time.deltaTime;
remainingTime = Mathf.Max(remainingTime, 0f);
```

更新 UI：

```csharp
int displaySeconds = Mathf.CeilToInt(remainingTime);

countdownText.text = displaySeconds.ToString();

countdownFill.fillAmount =
    gameplayDuration > 0f
        ? remainingTime / gameplayDuration
        : 0f;
```

不需要額外建立 `CountdownController`。

---

## 17.1 警示狀態

當：

```csharp
remainingTime <= warningStartTime
```

且警示尚未觸發時：

1. 將 `warningTriggered` 設為 true。
2. 觸發 `onCountdownWarning`。
3. 可設定 Animator Trigger。

每局只觸發一次。

---

## 17.2 時間歸零

當：

```csharp
remainingTime <= 0f
```

時：

1. 固定時間為 0。
2. 更新 UI 為 0。
3. 停止倒數。
4. 停止輸入。
5. 檢查結果是否已回報。
6. 呼叫失敗流程。

不得讓時間變成負數。

---

# 18. 空白鍵確認流程

`ItemSelectionController` 發出：

```csharp
ItemConfirmed
```

後，`GameplayController` 執行：

```csharp
HandleItemConfirmed(SelectableItem item);
```

首先檢查：

* 是否仍在 Gameplay
* `gameplayRunning` 是否為 true
* 結果是否尚未回報
* item 是否有效
* item 是否仍可選取
* item 是否尚未完成

---

# 19. 正確選擇流程

若：

```csharp
item.IsContraband == true
```

則：

1. 暫時停用確認輸入。
2. 呼叫 `item.MarkCorrect()`。
3. 增加 `foundContrabandCount`。
4. 取得 `item.ContrabandSlotIndex`。
5. 將對應 Slot 設為完成。
6. 觸發 Slot 的 `onCompleted`。
7. 觸發 `onCorrectSelection`。
8. 從目前可選取集合中排除該物件。
9. 自動選取下一個有效物件。
10. 判斷是否已找完全部違禁品。

完成條件：

```csharp
foundContrabandCount >= totalContrabandCount
```

若尚未完成：

* 恢復確認輸入
* 倒數繼續

---

# 20. 錯誤選擇流程

若：

```csharp
item.IsContraband == false
```

則：

1. 呼叫 `item.PlayWrongFeedback()`。
2. 觸發 `onWrongSelection`。
3. 不移除物件。
4. 不增加找到數量。
5. 不改變違禁品 UI。
6. 遊戲繼續。

如果目前關卡設定：

```csharp
wrongSelectionDeductsTime == true
```

則：

```csharp
remainingTime -= wrongSelectionTimePenalty;
remainingTime = Mathf.Max(remainingTime, 0f);
```

扣秒後立即更新 UI。

若扣秒後歸零，立即進入失敗流程。

---

# 21. 成功流程

找完全部違禁品後：

1. 確認結果尚未回報。
2. 將 `resultReported` 設為 true。
3. 將 `gameplayRunning` 設為 false。
4. 停止倒數。
5. 停止方向鍵與空白鍵輸入。
6. 清除目前選取效果。
7. 觸發 `onGameSuccess`。
8. 呼叫：

```csharp
gameFlowManager.NotifyGameSuccess();
```

不得由 `SelectableItem` 或 UnityEvent 直接呼叫成功。

成功後即使倒數在同一幀歸零，也不得再觸發失敗。

---

# 22. 失敗流程

時間歸零後：

1. 確認結果尚未回報。
2. 將 `resultReported` 設為 true。
3. 將 `gameplayRunning` 設為 false。
4. 停止方向鍵與空白鍵輸入。
5. 清除目前選取效果。
6. 觸發 `onGameFailure`。
7. 呼叫：

```csharp
gameFlowManager.NotifyGameFailure();
```

時間歸零後不得再接受空白鍵確認。

---

# 23. 最後瞬間判斷規則

為避免成功與失敗同時發生：

1. 玩家確認輸入時，先檢查 `remainingTime > 0`。
2. 若已等於 0，確認輸入無效。
3. 如果確認違禁品後已完成全部答案，立即鎖定成功。
4. 成功結果鎖定後，倒數不得再觸發失敗。
5. 所有成功與失敗流程都先檢查 `resultReported`。

`resultReported` 只能從 false 變成 true 一次。

---

# 24. 自動選取下一個物件

正確物件移除後：

1. 取得該物件移除前的螢幕座標。
2. 從剩餘可選取物件中找最近者。
3. 選取最近有效物件。
4. 若沒有剩餘物件，清除選取。

如果全部違禁品已完成：

* 不需要再選取普通物件
* 直接進入成功流程

---

# 25. 重置流程

進入 `Resetting` 時，`GameplayController` 執行：

1. 將 `gameplayRunning` 設為 false。
2. 停止輸入。
3. 清除目前選取。
4. 將倒數停止。
5. 將警示旗標清除。
6. 將結果旗標清除。
7. 將找到數量歸零。
8. 呼叫所有 `SelectableItem.ResetItem()`。
9. 恢復所有已隱藏物件。
10. 重置所有違禁品 Slot。
11. 隱藏 Gameplay UI。
12. 重置倒數 Animator。
13. 觸發 `onGameplayReset`。
14. 清除目前關卡 Runtime 集合。

不得 Destroy 關卡中原本存在的物件。

# 27. GameFlowManager UnityEvent 配置

沿用第二階段的 UnityEvent。

## onGameplayEntered

可綁定：

* 顯示 Gameplay UI
* 顯示操作提示
* 啟用 Gameplay 裝飾內容

核心 `BeginGameplay()` 建議透過 C# `StateChanged` 自動呼叫，不依靠 Inspector UnityEvent。

## onSuccessPreparingEntered

可綁定：

* 隱藏操作提示
* 播放成功轉場效果
* 停止警示音效

不可綁定：

```csharp
NotifyGameSuccess()
```

---

## onFailurePreparingEntered

可綁定：

* 隱藏操作提示
* 播放失敗轉場效果
* 停止警示音效

不可綁定：

```csharp
NotifyGameFailure()
```

---

## onResettingEntered

可綁定：

* 關閉共用 Gameplay 顯示
* 停止 AudioSource
* 停止 ParticleSystem
* 重置 Animator

核心物件與資料重置仍由 `GameplayController` 執行。

# 29. 驗收測試

## 29.1 Gameplay 初始化

1. 選擇 Level_01。
2. Intro 結束後進入 Gameplay。
3. 正確啟用 Level_01 Root。
4. 其他關卡 Root 保持停用。
5. 收集正確的可選物件。
6. 顯示倒數 UI。
7. 顯示違禁品提示 UI。
8. 自動選取預設物件。
9. 倒數從設定時間開始。
10. 方向鍵與空白鍵可以操作。

---

## 29.2 方向選取

確認：

* 上鍵選到畫面上方合理物件。
* 下鍵選到畫面下方合理物件。
* 左鍵選到畫面左側合理物件。
* 右鍵選到畫面右側合理物件。
* 不選到已完成物件。
* 不選到停用物件。
* 無候選時保持目前選取。
* 同一時間只有一個選取效果。
* 快速按鍵不會一次跳過多個物件。

---

## 29.3 正確選擇

1. 選取違禁品。
2. 按空白鍵。
3. 觸發該物件 `onCorrect`。
4. 物件設為不可再選取。
5. 對應 Slot 從剪影切換為彩色。
6. Slot 完成事件只觸發一次。
7. 找到數量增加。
8. 自動選取下一個有效物件。

---

## 29.4 錯誤選擇

1. 選取普通物件。
2. 按空白鍵。
3. 觸發該物件 `onWrong`。
4. 物件不消失。
5. 找到數量不增加。
6. 違禁品 UI 不改變。
7. 遊戲繼續。
8. 確認輸入冷卻有效。

---

## 29.5 倒數 UI

確認：

* 初始數字正確。
* 進度條與剩餘時間同步。
* 秒數使用向上取整。
* 最後設定秒數進入警示狀態。
* 警示事件每局只觸發一次。
* 時間不會顯示負數。
* 歸零後顯示 0。

---

## 29.6 成功

1. 在時間內找完全部違禁品。
2. 倒數立即停止。
3. 輸入立即停用。
4. 成功事件只觸發一次。
5. 呼叫 `NotifyGameSuccess()`。
6. 系統進入 SuccessPreparing。
7. 不會再觸發失敗。

---

## 29.7 失敗

1. 不完成全部違禁品。
2. 等待時間歸零。
3. 倒數 UI 顯示 0。
4. 輸入立即停用。
5. 失敗事件只觸發一次。
6. 呼叫 `NotifyGameFailure()`。
7. 歸零後不能再選取物件。

---

## 29.8 最後瞬間輸入

測試剩餘時間接近 0 時按空白鍵。

確認：

* 若有效確認先完成，判定成功。
* 若時間已歸零，確認無效並判定失敗。
* 不會同時觸發成功與失敗。
* `resultReported` 只會設定一次。

---

## 29.9 錯誤扣秒

當設定為啟用：

* 錯誤選擇扣除指定秒數。
* 倒數 UI 立即更新。
* 扣除後歸零可正常失敗。
* 時間不會變成負數。

當設定為關閉：

* 錯誤選擇不影響時間。

---

## 29.10 重置

完成一局後再次進入相同關卡。

確認：

* 所有物件重新顯示。
* 違禁品重新可選。
* 普通物件狀態正常。
* UI 回到剪影。
* 找到數量歸零。
* 倒數恢復初始時間。
* 警示動畫重置。
* 上一局選取效果不殘留。
* UnityEvent 不會每局增加一次。

---

## 29.11 六個關卡

逐一測試六個關卡：

* 啟用正確 Level Root。
* 收集正確物件數量。
* 違禁品數量正確。
* Slot 索引沒有超出範圍。
* 每關預設選取物件正確。
* 每關倒數設定正確。
* 上一關資料不會殘留。

# 30. 效能與簡化要求

必須遵守：

* 第三階段只新增三支主要腳本。
* 不額外建立多個 Coordinator 或 Manager。
* 不為違禁品 UI Slot 建立額外 MonoBehaviour。
* 不建立 Item ScriptableObject。
* 不建立另一套 Gameplay 狀態機。
* 不在 Update 中使用 `GameObject.Find`。
* 不在每幀重新收集全部物件。
* 只有方向輸入時才計算候選。
* 不在每次選取時建立新 Material。
* 不反覆 Instantiate 選取框。
* 不依 UI 狀態作為唯一成功判定。
* UnityEvent 只負責表現，不負責核心流程。
* `SelectableItem` 不可直接修改 GameState。
* `ItemSelectionController` 不可直接判定答案。
* `GameplayController` 不可修改影片播放架構。
