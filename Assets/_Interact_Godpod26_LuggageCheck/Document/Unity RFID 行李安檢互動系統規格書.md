# Unity RFID 行李安檢互動系統規格書

## 1. 專案概述

本專案為使用 Unity 製作的 RFID 行李安檢互動內容。

整體體驗由以下內容組成：

1. 待機循環影片
2. RFID 關卡選擇
3. 關卡進場影片
4. 行李箱物件選擇小遊戲
5. 倒數計時 UI
6. 成功或失敗影片分支
7. 關卡結束後返回待機畫面

使用者透過 RFID 感應選擇不同關卡，並在限定時間內，使用方向鍵與空白鍵從行李箱內找出所有違禁品。

本互動共有六組關卡。每一組關卡具有不同的行李物件配置、違禁品答案、提示 UI，以及成功與失敗影片。

---

# 2. 系統內容總覽

## 2.1 關卡數量

系統共包含六組關卡。

每組關卡包含：

* 一支關卡進場影片
* 一張進場影片最後一幀背景圖片
* 一支成功影片
* 一支失敗影片
* 一組行李內容物件
* 一組可選取物件資料
* 一組違禁品答案
* 一組違禁品提示 UI
* 一組正確與錯誤回饋特效
* 一組正確與錯誤音效

此外，整個系統共用一支待機 Loop 影片。

因此，完整影片數量暫定為：

| 影片類型       | 數量 |
| ---------- | -- |
| 待機 Loop 影片 | 1  |
| 關卡進場影片     | 6  |
| 成功影片       | 6  |
| 失敗影片       | 6  |
| 總計         | 19 |

---

## 2.2 各關卡共用規則

六個關卡使用相同的基本操作與流程，但以下內容會依照關卡個別設定：

* RFID 識別碼
* 關卡名稱與 ID
* 進場影片
* 成功影片
* 失敗影片
* 行李物件種類
* 行李物件數量
* 行李物件位置
* 違禁品種類
* 違禁品數量
* 違禁品提示 UI
* 預設選取物件
* 正確與錯誤特效
* 正確與錯誤音效
* 行李內容物件的移動動畫
* 遊戲倒數時間

遊戲倒數時間暫定為 15 秒，但必須製作成可調整參數。

---

# 3. 畫面圖層配置

畫面由下至上分為以下圖層：

1. 背景影片或靜態背景圖片
2. 行李箱內部的 Unity 互動物件
3. 物件選取提示
4. 物件正確或錯誤特效
5. 違禁品提示 UI
6. 倒數計時 UI
7. 操作提示或系統訊息 UI

背景影片固定顯示在最下層。

Unity 中的行李物件需疊加在影片中的行李箱畫面上，因此影片解析度、行李箱停止位置、Unity Camera、Canvas 與物件座標必須保持一致。

---

# 4. 整體互動流程

完整互動流程如下：

1. 系統啟動
2. 載入共用資源
3. 播放待機 Loop 影片
4. 等待 RFID 感應
5. 讀取 RFID 並決定關卡
6. 初始化對應關卡
7. 播放關卡進場影片
8. 行李內容物件跟隨影片中的行李箱移動
9. 進場影片播放結束
10. 背景切換為影片最後一幀圖片
11. 顯示並啟用倒數計時 UI
12. 啟用方向鍵與空白鍵操作
13. 玩家尋找違禁品
14. 判斷成功或失敗
15. 播放成功或失敗影片
16. 影片播放完成
17. 清除關卡狀態
18. 返回待機 Loop 影片
19. 等待下一次 RFID 感應

---

# 5. 系統狀態設計

建議使用有限狀態機管理整個流程。

| 狀態                 | 說明                   |
| ------------------ | -------------------- |
| SystemInitializing | 系統啟動與基本資源初始化         |
| IdlePreparing      | 準備待機影片               |
| Idle               | 播放待機 Loop 影片並等待 RFID |
| LevelInitializing  | 載入及重置對應關卡            |
| IntroPreparing     | 預載關卡進場影片             |
| IntroPlaying       | 播放行李箱進場影片            |
| GamePreparing      | 將影片切換為靜態圖片並準備互動      |
| Gameplay           | 倒數計時並接受玩家操作          |
| SuccessPreparing   | 停止操作並預備成功影片          |
| SuccessPlaying     | 播放成功影片               |
| FailurePreparing   | 停止操作並預備失敗影片          |
| FailurePlaying     | 播放失敗影片               |
| Resetting          | 清除目前關卡狀態             |
| ErrorRecovery      | 處理影片或硬體異常            |

不同狀態只能執行該狀態允許的操作，避免 RFID、鍵盤輸入或影片事件重複觸發。

---

# 6. 待機狀態

## 6.1 待機影片

系統完成初始化後，播放一支循環影片作為待機畫面。

待機影片需設定為：

* 自動循環播放
* 播放完成後不出現黑畫面
* 不顯示關卡物件
* 不啟動倒數計時
* 不接受方向鍵及空白鍵操作
* 只接受有效的 RFID 輸入

待機影片播放期間，系統持續等待 RFID 感應。

---

## 6.2 RFID 感應

使用者將 RFID 卡片或感應物靠近讀取裝置後，RFID 裝置將識別碼傳送給 Unity。

Unity 接收到資料後需執行：

1. 讀取 RFID 原始字串
2. 移除換行、空白及無效字元
3. 比對 RFID 與關卡對應資料
4. 確認目前系統處於 Idle 狀態
5. 若 RFID 有效，進入對應關卡
6. 若 RFID 無效，忽略該次感應
7. 關卡開始後鎖定 RFID 輸入

RFID 對應關係範例如下：

| RFID 編號 | 對應關卡 |
| ------- | ---- |
| RFID_01 | 關卡 1 |
| RFID_02 | 關卡 2 |
| RFID_03 | 關卡 3 |
| RFID_04 | 關卡 4 |
| RFID_05 | 關卡 5 |
| RFID_06 | 關卡 6 |

需避免卡片停留在感應區時重複觸發。

可使用以下機制：

* 同一張卡片的觸發冷卻時間
* 關卡執行期間停止讀取 RFID
* 必須先偵測卡片移開，才能再次觸發
* 記錄最後一次 RFID 編號及觸發時間

---

# 7. 關卡初始化

RFID 決定關卡後，進入關卡初始化流程。

系統需執行：

* 暫停新的 RFID 輸入
* 停止待機影片
* 載入對應關卡資料
* 啟用該關卡的行李物件群組
* 隱藏其他關卡物件
* 重置所有物件狀態
* 重置所有物件位置
* 重置物件選取狀態
* 重置已找到的違禁品數量
* 重置違禁品提示 UI
* 隱藏倒數計時 UI
* 重置倒數秒數
* 停止上一局殘留的音效
* 清除上一局殘留的粒子特效
* 準備進場影片

初始化未完成前，不允許進入互動階段。

---

# 8. 關卡進場影片

## 8.1 影片內容

關卡開始時播放該關卡的進場影片。

影片內容為：

* 輸送帶運送一個空的行李箱
* 行李箱由畫面左側移動至中央
* 行李箱最後停在互動位置

影片中的行李箱本體由影片呈現。

行李箱內的內容物則由 Unity 中的 2D 或 3D 物件呈現，並疊加在影片上方。

---

## 8.2 行李物件同步移動

進場影片播放時，Unity 中的行李內容物件群組需要同步跟隨影片中的行李箱移動。

行李物件群組需從畫面左側移動至中央，並在影片結束時準確停在行李箱內。

同步方式可使用：

* Unity Timeline
* Animator
* Animation Clip
* 自訂 AnimationCurve
* 根據 VideoPlayer目前播放時間計算位置

建議使用影片目前播放時間或目前播放幀作為同步基準，而不是讓影片與物件各自獨立計時。

物件位置可根據影片播放進度計算：

* 影片播放進度為 0 時，物件位於起始位置
* 影片播放進度為 1 時，物件位於互動位置
* 中間位置依照指定的動畫曲線進行插值

如果影片因裝置效能而短暫延遲，Unity 物件應依影片當前時間重新計算位置，避免影片與物件逐漸不同步。

---

## 8.3 進場期間輸入限制

進場影片播放期間：

* 不接受方向鍵操作
* 不接受空白鍵操作
* 不啟動倒數計時
* 不允許 RFID 切換關卡
* 不顯示選取提示
* 不允許行李物件被確認
* 倒數計時 UI保持隱藏

---

# 9. 進場影片與遊戲畫面切換

進場影片播放完成後，背景需從影片切換為該影片最後一幀的靜態圖片。

流程如下：

1. 確認進場影片確實播放至結尾
2. 將 Unity 行李物件固定在最終位置
3. 顯示與影片最後一幀完全一致的背景圖片
4. 確認背景圖片已顯示
5. 隱藏或停止目前影片輸出
6. 啟用互動物件
7. 顯示物件選取提示
8. 顯示違禁品提示 UI
9. 顯示倒數計時 UI
10. 啟動倒數計時
11. 開放方向鍵與空白鍵輸入

影片與靜態圖片的切換不得出現：

* 黑畫面
* 白畫面
* 單幀閃爍
* 畫面位置偏移
* 縮放比例變化
* 色彩或亮度明顯跳動
* 行李箱位置跳動

靜態背景圖片需直接由進場影片的最後一幀輸出，並使用相同解析度及色彩設定。

切換時應先顯示靜態圖片，再關閉影片畫面，避免兩者之間出現空白幀。

---

# 10. 倒數計時 UI

## 10.1 基本需求

互動小遊戲開始後，畫面需顯示倒數計時 UI。

倒數時間暫定為 15 秒。

倒數計時必須是關卡可調整參數，不應直接寫死在程式內。

倒數計時 UI 至少包含：

* 剩餘秒數文字
* 倒數進度圖形
* 時間即將結束時的警示狀態

建議同時使用數字與進度圖形，讓使用者能快速理解剩餘時間。

---

## 10.2 顯示形式

倒數計時 UI 可由以下元素組成：

* 大型倒數秒數
* 圓形倒數進度
* 水平進度條
* 時間圖示
* 最後數秒的警示動畫

例如：

| 剩餘時間   | UI 狀態          |
| ------ | -------------- |
| 15～6 秒 | 一般顯示狀態         |
| 5～4 秒  | 進入警示狀態         |
| 3～1 秒  | 數字放大、閃爍並播放倒數音效 |
| 0 秒    | 停止操作並進入失敗流程    |

實際警示秒數應製作為可調整參數。

---

## 10.3 倒數計時規則

倒數計時從 Gameplay 狀態正式開始後才啟動。

以下階段不計時：

* 待機畫面
* RFID 感應
* 關卡初始化
* 影片載入
* 進場影片播放
* 影片與圖片切換
* 成功影片播放
* 失敗影片播放
* 系統重置

Gameplay 狀態開始時：

1. 將剩餘時間設為關卡設定時間
2. 顯示倒數計時 UI
3. 開始減少時間
4. 每幀更新數字及進度圖形
5. 時間歸零時停止玩家輸入
6. 判斷為失敗

倒數顯示建議使用向上取整。

例如剩餘時間為 14.2 秒時，UI 顯示 15 秒；剩餘時間為 0.2 秒時，UI 顯示 1 秒。

如此可避免遊戲開始後立即從 15 跳成 14，或在畫面顯示 0 秒時遊戲仍短暫可以操作。

---

## 10.4 倒數結束判斷

為避免最後一刻按鍵與倒數結束同時發生，成功與失敗判斷必須由同一個遊戲控制系統管理。

建議判斷順序：

1. 接收到玩家確認輸入
2. 判斷目前是否仍在 Gameplay 狀態
3. 判斷剩餘時間是否大於 0
4. 處理正確或錯誤選擇
5. 若已找到全部違禁品，立即判定成功
6. 若尚未完成，再持續更新時間
7. 當剩餘時間小於或等於 0 時判定失敗

一旦判定成功或失敗，Gameplay 狀態立即結束，避免同一幀同時觸發成功與失敗。

---

# 11. 違禁品提示 UI

畫面下方設置違禁品提示區域。

每個關卡需要找出的違禁品，各自對應一個提示欄位。

## 11.1 尚未找到

尚未找到時，UI 顯示：

* 違禁品剪影
* 灰色或暗色圖示
* 未完成狀態
* 必要時顯示剩餘數量

---

## 11.2 已找到

正確找到違禁品後，對應 UI 欄位需：

* 將剪影替換為彩色物件圖片
* 播放完成特效
* 播放對應音效
* 顯示已完成狀態
* 禁止相同項目重複計算

完成狀態需保留到本關卡結束。

---

# 12. 物件選取操作

## 12.1 操作方式

| 按鍵   | 功能        |
| ---- | --------- |
| 上方向鍵 | 選擇上方物件    |
| 下方向鍵 | 選擇下方物件    |
| 左方向鍵 | 選擇左側物件    |
| 右方向鍵 | 選擇右側物件    |
| 空白鍵  | 確認目前選取的物件 |

只有在 Gameplay 狀態下接受上述輸入。

---

## 12.2 不規則物件選擇

行李箱內物件並非規則排列，因此不能單純使用固定的上下左右索引。

方向選擇應根據畫面空間中的相對方向及距離，找出最合理的目標。

候選物件需符合：

* 位於輸入方向的有效範圍
* 尚未被正確移除
* 目前仍可選取
* 距離及方向角度合理

候選物件評分可考慮：

* 與輸入方向的夾角
* 與目前物件的距離
* 水平或垂直方向偏移
* 是否被其他物件遮擋
* 是否屬於有效選取區域

方向越接近輸入方向、距離越近的物件，應具有越高優先度。

---

## 12.3 選取提示

同一時間只能有一個物件處於選取狀態。

選取提示可以使用：

* 發光外框
* 選取框
* 箭頭
* 脈衝縮放
* 色彩變化
* Shader 效果
* 物件上方圖示

遊戲開始時必須自動選取一個預設物件。

預設選取物件可由各關卡資料指定。

若目前物件被正確移除，系統需自動尋找下一個可選取物件。

---

# 13. 正確選擇流程

當玩家按下空白鍵，且目前選取的是違禁品時：

1. 暫時鎖定確認輸入
2. 將該物件標記為已找到
3. 停止該物件繼續被選取
4. 在物件位置播放正確特效
5. 播放正確音效
6. 將違禁品從行李箱中移除
7. 更新下方對應的提示 UI
8. 將 UI 剪影替換為彩色圖片
9. 在 UI 欄位播放完成特效
10. 更新已找到違禁品數量
11. 判斷是否已找到全部違禁品
12. 若尚未完成，自動選取下一個物件
13. 解除輸入鎖定

違禁品移除方式可依視覺設計選擇：

* 直接隱藏
* 淡出
* 縮小後消失
* 飛向下方提示 UI
* 播放指定動畫後消失

---

# 14. 錯誤選擇流程

當玩家按下空白鍵，且目前選取的不是違禁品時：

1. 該物件保留在行李箱中
2. 在物件位置播放錯誤特效
3. 播放錯誤音效
4. 可播放短暫震動或縮放動畫
5. 倒數計時持續進行
6. 遊戲繼續
7. 該物件仍可再次被選取

為避免連續按鍵造成特效及音效大量重疊，需加入短暫輸入冷卻。

錯誤選擇是否扣除時間，應保留為可設定參數。

預設規則為不扣時間。

---

# 15. 成功條件與流程

當玩家在時間歸零前找到全部違禁品，判定為成功。

成功流程：

1. 將系統狀態切換為 SuccessPreparing
2. 停止倒數計時
3. 隱藏倒數計時 UI，或顯示成功狀態
4. 停止接受方向鍵及空白鍵
5. 移除物件選取提示
6. 完成最後一個物件的正確回饋
7. 顯示成功影片的第一幀或準備遮罩
8. 確認成功影片已完成預載
9. 將靜態背景切換為成功影片
10. 播放成功影片
11. 成功影片播放完畢後進入 Resetting
12. 清除關卡狀態
13. 返回待機 Loop 影片

成功後不得因倒數計時的殘留更新而進入失敗流程。

---

# 16. 失敗條件與流程

當倒數時間歸零，且仍有違禁品尚未找到時，判定為失敗。

失敗流程：

1. 將系統狀態切換為 FailurePreparing
2. 將剩餘時間固定為 0
3. 停止倒數計時
4. 停止接受方向鍵及空白鍵
5. 移除物件選取提示
6. 顯示失敗狀態或隱藏倒數 UI
7. 確認失敗影片已完成預載
8. 將靜態背景切換為失敗影片
9. 播放失敗影片
10. 失敗影片播放完畢後進入 Resetting
11. 清除關卡狀態
12. 返回待機 Loop 影片

---

# 17. StreamingAssets 影片管理

## 17.1 影片存放位置

所有影片檔案存放於 Unity 專案的以下資料夾：

`Assets/StreamingAssets/LuggageCheck/Videos/`

建議資料夾結構如下：

```text
Assets/
└── StreamingAssets/
    └── LuggageCheck/
        └── Videos/
            ├── Common/
            │   └── Idle_Loop.mp4
            │
            ├── Level01/
            │   ├── Level01_Intro.mp4
            │   ├── Level01_Success.mp4
            │   └── Level01_Failure.mp4
            │
            ├── Level02/
            │   ├── Level02_Intro.mp4
            │   ├── Level02_Success.mp4
            │   └── Level02_Failure.mp4
            │
            └── Level06/
                ├── Level06_Intro.mp4
                ├── Level06_Success.mp4
                └── Level06_Failure.mp4
```

靜態背景圖片需與影片一樣存放於 StreamingAssets 中，並透過相對路徑由關卡資料引用，以確保在 Build 後仍可正確讀取與顯示。

---

## 17.2 影片路徑

影片路徑由以下方式組合：

```csharp
Path.Combine(
    Application.streamingAssetsPath,
    relativeVideoPath
);
```

關卡資料不應保存完整的電腦絕對路徑，只需保存相對路徑。

例如：

```text
LuggageCheck/Videos/Level01/Level01_Intro.mp4
```

不可將以下形式直接寫入關卡資料：

```text
C:/Project/MyUnityProject/Assets/StreamingAssets/...
```

以避免程式換電腦或輸出 Build 後路徑失效。

---

## 17.3 StreamingAssets 注意事項

StreamingAssets 中的影片會以原始檔案形式包含在輸出程式中，適合使用 VideoPlayer URL 模式載入。

但 StreamingAssets 不代表影片一定能立即播放。

影片第一次開啟時仍可能需要：

* 建立檔案存取
* 開啟影片解碼器
* 讀取影片標頭
* 準備第一幀
* 建立 RenderTexture
* 建立音訊輸出
* 等待儲存裝置讀取

因此，不可在呼叫播放的同一幀直接假設影片已準備完成。

所有影片播放前，都必須先經過 Prepare 流程。

---

# 18. VideoPlayer 播放架構

## 18.1 基本設定

影片建議使用 Unity VideoPlayer 播放。

VideoPlayer 建議設定：

* Source：URL
* URL：StreamingAssets 影片路徑
* Render Mode：Render Texture
* Play On Awake：關閉
* Wait For First Frame：開啟
* Skip On Drop：依同步需求測試
* Audio Output Mode：Direct 或 AudioSource
* Is Looping：只有待機影片開啟
* Controlled Audio Track Count：依影片規格設定

背景影片輸出至 RenderTexture，再由 RawImage 或指定材質顯示在畫面最底層。

---

## 18.2 影片預載

播放每支影片前需執行：

1. 設定 VideoPlayer URL
2. 呼叫 `Prepare()`
3. 等待 `prepareCompleted`
4. 確認 `isPrepared` 為 true
5. 確認影片畫面已準備
6. 才允許切換並播放

在準備完成前：

* 保持目前背景顯示
* 不顯示未準備完成的 VideoPlayer
* 不直接清空上一個背景
* 不進入需要影片播放的狀態

---

## 18.3 進場影片預載時機

RFID 感應後，系統立即開始準備該關卡的進場影片。

準備期間：

* 待機影片可繼續顯示
* 或使用待機影片最後一幀作為過渡
* 關卡物件保持隱藏
* 不啟動計時
* 不接受鍵盤操作

當進場影片準備完成後，才停止待機畫面並開始播放進場影片。

不可先停止待機影片，再等待進場影片載入，否則容易出現黑畫面。

---

## 18.4 成功與失敗影片預載時機

為避免成功或失敗判定後才開始載入影片，造成明顯停頓，建議在互動遊戲開始前或進場影片播放期間，預先準備成功與失敗影片。

由於最終只會播放其中一支，可採用以下架構之一：

### 方案 A：三個 VideoPlayer

每個用途使用獨立播放器：

* Idle VideoPlayer
* Intro VideoPlayer
* Result VideoPlayer

互動遊戲開始時，Result VideoPlayer 預先準備成功影片。

若結果為失敗且失敗影片尚未準備，則需切換 URL 並 Prepare。

此方案記憶體使用較低，但失敗分支可能需要額外等待。

### 方案 B：雙結果播放器

分別準備：

* Success VideoPlayer
* Failure VideoPlayer

互動遊戲開始時同時預載成功與失敗影片。

結果確定後，可直接顯示對應播放器並播放。

此方案切換最順暢，但會使用較多解碼器及記憶體。

### 方案 C：共用播放器依序預載

進場影片播放期間先準備成功影片，成功影片完成後再準備失敗影片。

若兩者都能在互動開始前準備完成，即可降低切換延遲。

最終採用方案需依目標電腦的記憶體、顯示卡與影片解析度進行壓力測試。

若硬體允許，建議優先確保成功與失敗兩個分支都已準備完成。

---

# 19. 無縫影片切換

## 19.1 切換基本原則

任何影片切換都必須遵循：

1. 下一個畫面先準備完成
2. 下一個畫面先顯示
3. 確認下一個畫面有效
4. 再隱藏上一個畫面

不可採用：

1. 先停止目前影片
2. 清空 RenderTexture
3. 再開始載入下一支影片

此流程容易產生黑畫面。

---

## 19.2 待機影片切換至進場影片

順序如下：

1. 待機影片持續播放
2. 背景層保持顯示
3. 進場 VideoPlayer 執行 Prepare
4. 等待進場影片準備完成
5. 將進場影片定位至起始畫面
6. 顯示進場影片輸出
7. 開始播放進場影片
8. 隱藏或暫停待機影片

---

## 19.3 進場影片切換至靜態背景

順序如下：

1. 預先載入最後一幀背景圖片
2. 進場影片播放至結尾
3. 將 Unity 行李物件固定在最終位置
4. 顯示最後一幀背景圖片
5. 確認圖片已顯示
6. 隱藏影片 RawImage
7. 開始互動遊戲

---

## 19.4 靜態背景切換至結果影片

順序如下：

1. 成功或失敗影片預載完成
2. 結果影片定位至第一幀
3. 顯示影片輸出
4. 開始播放
5. 隱藏靜態背景圖片
6. 關閉互動物件及遊戲 UI

成功與失敗影片的第一幀應盡量與遊戲結束畫面一致，以降低畫面跳動感。

---

## 19.5 結果影片切換回待機影片

在結果影片播放期間，可提前準備待機影片。

結果影片結束後：

1. 確認待機影片已準備完成
2. 顯示待機影片第一幀
3. 開始播放待機影片
4. 隱藏結果影片
5. 清除上一關卡物件及 UI
6. 重新開放 RFID 感應

---

# 20. 載入畫面與等待處理

正常情況下，使用者不應看到明顯載入畫面。

影片準備期間應維持上一個有效畫面，例如：

* RFID 感應後，待機 Loop 繼續播放
* 結果影片未準備完成前，維持遊戲最後畫面
* 待機影片未準備完成前，維持結果影片最後一幀

若影片準備時間超過指定上限，可顯示簡單載入提示，但不可直接顯示黑畫面。

載入逾時時間應製作成設定值，例如 5 秒或 10 秒。

# 21. 影片播放事件

系統需監聽 VideoPlayer 事件：

* `prepareCompleted`
* `loopPointReached`
* `errorReceived`
* 必要時使用 `frameReady`

用途如下：

| 事件               | 用途              |
| ---------------- | --------------- |
| prepareCompleted | 確認影片可播放         |
| loopPointReached | 判斷非 Loop 影片播放完成 |
| errorReceived    | 處理影片路徑、格式或解碼錯誤  |
| frameReady       | 需要精確取得畫面幀時使用    |

不可只依照預估影片長度使用 Coroutine 等待，因為實際播放可能受到硬體效能或讀取速度影響。

影片結束應以 VideoPlayer 播放事件或實際播放狀態為主要判斷依據。

---

# 22. 影片編碼與輸出規格

所有影片應統一：

* 解析度
* 畫面比例
* 幀率
* 編碼
* 音訊格式
* 色彩空間
* 音量標準
* 行李箱最終位置

建議規格範例：

| 項目   | 建議                        |
| ---- | ------------------------- |
| 容器格式 | MP4                       |
| 影片編碼 | H.264                     |
| 幀率   | 所有影片一致，例如 30 fps 或 60 fps |
| 解析度  | 依最終螢幕輸出解析度                |
| 音訊   | AAC，48 kHz                |
| 關鍵影格 | 避免間隔過長                    |
| 位元率  | 在畫質與讀取效能間取得平衡             |

影片位元率不可過高，否則即使影片位於本機，也可能造成：

* 開啟影片時間過長
* 第一幀等待
* 播放掉幀
* 硬碟讀取壓力
* 解碼器壓力
* 多影片預載時記憶體增加

所有影片應在目標展示電腦上實際測試。

* 清除所有物件選取狀態

# 26. 異常處理

## 26.1 影片不存在

若 StreamingAssets 中找不到指定影片：

* 記錄缺少的影片路徑
* 不進入無限等待
* 顯示備用背景
* 返回待機狀態
* 重新開放 RFID

---

## 26.2 影片準備失敗

若收到 VideoPlayer 的錯誤事件：

* 記錄錯誤內容
* 停止目前播放流程
* 保持上一個有效畫面
* 顯示必要的系統錯誤提示
* 嘗試返回待機狀態

---

## 26.3 影片載入逾時

若影片在設定時間內未完成 Prepare：

* 結束等待
* 記錄逾時的影片
* 不顯示未準備完成的影片
* 返回待機或播放備用內容

---

## 26.4 RFID 無效

若 RFID 不存在於關卡對應表：

* 忽略輸入
* 保持待機影片播放
* 不進入載入狀態
* 可選擇播放無效感應提示音

---

## 26.5 無可選取物件

若 Gameplay 開始時沒有可選取物件：

* 不啟動倒數
* 記錄關卡資料錯誤
* 返回待機狀態

---

## 26.6 遊戲中斷

若程式失去焦點、影片停止或硬體暫時中斷，需依展示需求決定：

* 暫停遊戲倒數
* 繼續倒數
* 直接重置回待機

建議展場模式下，發生無法恢復的異常時自動返回待機，避免系統停在不可操作畫面。

---

# 27. 效能與穩定性需求

系統應符合以下要求：

* 待機影片可長時間循環播放
* 連續遊玩不同關卡不產生記憶體持續增加
* 影片播放完成後正確釋放不再使用的資源
* RenderTexture 不重複建立
* 不在每次關卡開始時重新 Instantiate 大量固定物件
* 六組關卡物件可使用預先建立或物件池管理
* 粒子特效播放完成後正確停止及重置
* AudioSource 不持續累積
* 不因影片切換產生黑畫面
* 不因影片 Prepare 阻塞 Unity 主執行緒
* 不在 Update 中反覆組合檔案路徑或尋找物件
* 關卡重複進入後，結果必須與第一次遊玩一致

# 29. 驗收條件

## 29.1 待機與 RFID

* 系統啟動後可正常播放待機 Loop 影片
* 待機影片可長時間循環播放
* 待機期間方向鍵及空白鍵無效
* 六組 RFID 均能進入正確關卡
* 無效 RFID 不會進入關卡
* RFID 持續停留時不會重複觸發
* 關卡進行中不接受新的 RFID

---

## 29.2 StreamingAssets 與影片載入

* Build 後可從 StreamingAssets 正確讀取所有影片
* 影片路徑不依賴專案所在的絕對路徑
* 每支影片播放前均執行 Prepare
* 影片未準備完成前不會直接切換畫面
* 找不到影片時不會造成程式卡死
* Prepare 失敗時可返回待機狀態
* 影片切換時不出現明顯黑畫面
* 長時間運行後影片仍可正常播放

---

## 29.3 進場流程

* RFID 感應後播放正確的進場影片
* 進場影片準備期間待機畫面持續顯示
* Unity 行李內容物件與影片行李箱同步移動
* 影片延遲時物件不會持續累積位置誤差
* 進場期間不能操作物件
* 進場期間倒數計時不會啟動
* 影片結束後可無縫切換至最後一幀圖片
* 影片與圖片切換時位置及亮度無明顯跳動

---

## 29.4 倒數計時 UI

* 遊戲開始時顯示倒數計時 UI
* 初始時間正確顯示為關卡設定值
* 倒數數字與進度圖形同步
* 倒數不會在影片播放期間提前開始
* 最後數秒會進入警示狀態
* 時間歸零時停止接受輸入
* 成功後倒數立即停止
* 成功與失敗不會在同一幀同時觸發
* 返回待機後倒數 UI 正確重置

---

## 29.5 遊戲操作

* 遊戲開始時自動選取預設物件
* 方向鍵可在不規則排列物件間移動選取
* 同一時間只有一個選取提示
* 空白鍵可確認目前選取物件
* 正確物件會播放指定特效與音效
* 正確物件會從行李箱中消失
* 下方對應剪影會切換為彩色圖片
* 錯誤物件不會消失
* 錯誤選擇會播放錯誤特效與音效
* 正確物件不會重複計算
* 移除物件後能自動選取下一個有效物件

---

## 29.6 成功與失敗影片

* 找完全部違禁品時立即判定成功
* 成功後停止所有操作
* 成功影片已準備完成後才切換
* 成功影片播放完成後返回待機
* 時間歸零且未完成時判定失敗
* 失敗後停止所有操作
* 失敗影片已準備完成後才切換
* 失敗影片播放完成後返回待機
* 結果影片與前一畫面切換時不出現黑畫面

---

## 29.7 重置與長時間運行

* 關卡結束後所有物件恢復初始狀態
* 違禁品提示 UI 恢復為剪影
* 倒數時間恢復為初始值
* 上一關的粒子與音效不會殘留
* 可連續進入相同關卡
* 可連續進入不同關卡
* 多次遊玩後不發生狀態錯亂
* 多次播放後記憶體不持續明顯增加
* 發生影片錯誤後系統可恢復至待機狀態
